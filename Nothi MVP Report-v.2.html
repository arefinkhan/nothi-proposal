Executive Summary

This document outlines an aggressive 3-month (12-week) plan to deliver a Minimum Viable Product (MVP) for the Nothi Modernization project. The primary goal is to solve the platform's critical performance and usability issues for the 10 most common workflows.

We will build a new, high-performance foundation on your on-premises Oracle VM infrastructure using a modern, container-based (Kubernetes) architecture. This MVP will serve a pilot group of users and prove the new system's stability, speed, and value, building essential user trust for future phases.

All other features (AI assistant, All workflows) are on the "Future Roadmap."

MVP Scope & Key Deliverables

This 3-month project is laser-focused on what we can build, test, and deploy successfully.

Key Deliverables (In Scope)

On-Premises Production Infrastructure:

  • Fully configured Kubernetes (K8s) cluster on Oracle VMs.
  • High-availability Oracle Database instance.
  • On-premises object storage (MinIO).
  • On-premises message queue (RabbitMQ).
  • Secured API Gateway (Kong) & Identity (Keycloak) integrated with AD/LDAP.

Core Workflow MVP Application:

  • Fast, responsive web UI optimized for low-bandwidth.
  • Backend microservices for ten (10) complete workflows.

Pilot User Go-Live:

  • Successful deployment for 50-100 pilot users.
  • Training materials and support for pilot group.

Out of Scope (Phase 2)

To succeed, we must be ruthless about scope. The following are deferred to the Phase 2 roadmap:

  • NO AI Policy Advisor
  • NO Low-code visual builder
  • NO Migration of all workflows
  • NO Full data migration

The 12-Week Detailed Plan

We will run six 2-week sprints, with a heavy emphasis on parallel development in Month 2.

WEEKS 1-4

Month 1: Foundation

Goal: Build the "scaffolding" and finalize all specs.

  • Finalize 10 MVP workflows
  • Provision K8s, DB, MinIO
  • CI/CD pipeline functional
  • Login & Core Services (Auth, Doc)
WEEKS 5-8

Month 2: Parallel Build

Goal: Heads-down coding on all 10 workflows.

  • Backend logic for 10 workflows
  • Frontend UI for all 10 flows
  • End-to-end integration
  • First end-to-end tests
WEEKS 9-12

Month 3: Harden & Go-Live

Goal: Stabilize, test, and deploy to pilot users.

  • Intensive bug fixing
  • Performance & regression testing
  • User Acceptance Testing (UAT)
  • Pilot Go-Live & Hypercare

Key Risks & Mitigation

Risk: Scope Creep

Likelihood: VERY HIGH

Mitigation: A strict "No" policy is in effect. All new requests are added to the "Phase 2 Roadmap" backlog. The PM will relentlessly defend this scope.

Risk: Infrastructure Delays

Likelihood: HIGH

Mitigation: DevOps and DBA teams start on Day 1. We must have full access to the Oracle VM environment immediately.

Risk: Team Burnout

Likelihood: HIGH

Mitigation: This 3-month timeline is hyper-aggressive. We must ruthlessly de-scope any "gold plating" and focus only on the minimal viable path.

On-Premises Technical Architecture

We will adapt modern cloud-native concepts to run entirely within your data center on Oracle VM.

Host & Orchestration

  • Host: Government Data Center (GDC)
  • Virtualization: Oracle VM (OVM)
  • Orchestration: Kubernetes (K8s) Cluster

Data Tier (On-Prem)

  • Database: Oracle (High Availability)
  • Storage: MinIO (S3-Compatible)
  • Messaging: RabbitMQ (in K8s)

Security & Ingress

  • Gateway: Kong (On-premises)
  • Identity: Keycloak (in K8s)
  • Integration: Gov LDAP/Active Directory

Application Tier (Microservices)

  • Authentication MS
  • Config Server
  • Document Upload MS
  • Permissions MS
  • Nothi Core (x10 Workflows)
  • Email Service (Internal SMTP)
